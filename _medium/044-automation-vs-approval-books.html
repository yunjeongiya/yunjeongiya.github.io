Should the System Decide, or Should a Human Confirm? — Books an AI Recommended

I'm building a SaaS for managing after-school academies (학원) in Korea. One question keeps haunting every feature I build:

"Should the system handle this automatically, or should the teacher confirm it first?"

For example:

  • Should invoices be generated automatically each month, or should a teacher review and approve them first?
  • When a student is late, should the system immediately notify the parent, or should the teacher decide whether to send the notification?
  • When a sibling withdraws, should the sibling discount be removed automatically, or should we ask the teacher first?

This question follows every single feature. Automate too much, and a wrong notification goes to a parent. Add too many confirmations, and teachers drown in approval fatigue.

I'd been making these decisions by gut feeling — "this one should be automatic," "this one needs confirmation." No consistent framework. So I asked Claude (AI): "Are there any books I should read about this?"

Books the AI Recommended

Directly Relevant

1. Humans and Automation — Thomas Sheridan

The foundational text on automation research. Famous for the "10 levels of automation" model, ranging from Level 1 (human decides everything) to Level 10 (system does everything, ignores human). Instead of thinking "automatic or manual?" as a binary choice, this framework lets you ask: "Where on the 10-level spectrum should this feature sit?"

2. About Face: The Essentials of Interaction Design — Alan Cooper

Often called the bible of interaction design. Covers user control, approval flows, and modal vs. modeless design systematically. Essential reference when debating whether to add a confirmation dialog.

3. Don't Make Me Think — Steve Krug

Short and practical. Every approval step you add is another moment where the user has to think. This book helps distinguish between confirmations that genuinely protect the user and ones that just add friction.

Indirectly Relevant

4. The Design of Everyday Things — Don Norman

The textbook on affordances, feedback, and error prevention. The foundation for "let the system act, but design it to prevent mistakes."

5. Designing with the Mind in Mind — Jeff Johnson

Cognitive psychology-based UI design. Helps understand why users experience approval fatigue and what kinds of automation feel reassuring vs. threatening.

Key Paper (Free)

Parasuraman, Sheridan & Wickens (2000) — "A Model for Types and Levels of Human Interaction with Automation"

Published in IEEE Transactions. Splits automation into four functions (information acquisition → analysis → decision selection → action implementation) and defines 10 levels for each. The most practical and directly applicable framework. Available at PhilPapers.

A Decision Tree We Made Along the Way

I haven't read these books yet. But while discussing with the AI, we built a decision tree tailored to our system:

Does it send something directly to a parent?
  → YES → Preview + teacher confirmation before sending
  → NO ↓

Does it affect money? (invoices, discounts, refunds)
  → YES → Teacher confirmation required (at least 1-click approval)
  → NO ↓

Is it easily reversible?
  → NO → Teacher confirmation required
  → YES ↓

Is it a high-frequency routine task?
  → YES → Auto-execute + post-hoc review
  → NO → Teacher executes manually

And at every level: provide a per-campus override option. Different academies have different management styles. Some principals want "just handle everything for me." Others want "I'll check everything myself."
